Panorama de Plataformas de Datos

Bases de Datos Populares: un Mapa por Familias

16 plataformas agrupadas por lo que realmente las diferencia — no por marca, sino por el problema que resuelven.

Relacionales (SQL)

MySQLCódigo abierto, la más usada en apps web
PostgreSQLExtensible, soporte geoespacial y texto completo
SQL ServerMicrosoft, integrado al ecosistema Windows
Oracle DatabaseAlto rendimiento, entornos empresariales
SQLiteSin servidor, embebida en apps y navegadores
MariaDBFork de MySQL, foco en comunidad y seguridad
Ideal para: transacciones, ERPs, sistemas financieros — donde el esquema fijo y las relaciones entre tablas son la regla, no la excepción.

NoSQL

MongoDBDocumentos, flexible y escalable
RedisEn memoria — caché, colas, tiempo real
CassandraDistribuida, escrita para alta disponibilidad
CouchbaseDocumentos + memoria en un mismo motor
Ideal para: catálogos de producto, IoT, datos que cambian de forma — volumen y escala horizontal antes que esquema rígido.

Cloud-managed

Amazon AuroraCompatible MySQL/PostgreSQL, AWS
Cloud SpannerRelacional distribuida, consistencia global
Azure SQL DatabaseRelacional administrada, alta disponibilidad
Firebase Realtime DBSincronización en tiempo real, móvil/web
Ideal para: equipos sin DBA dedicado — el proveedor gestiona parcheo, backups y escalado.

Especializadas

Neo4jGrafos — datos y relaciones complejas
ElasticsearchBúsqueda y análisis en tiempo real
Apache HBaseNoSQL sobre Hadoop, grandes volúmenes
Ideal para: recomendaciones, redes sociales, búsqueda de logs — un caso de uso puntual, no un motor de propósito general.
Cómo elegir — relacional cuando la integridad y las relaciones entre tablas importan (ACID, joins); NoSQL cuando el volumen y la escala horizontal pesan más que el esquema fijo; cloud-managed cuando prefieres ceder la operación de infraestructura; especializada cuando el caso de uso es puntual (grafos, búsqueda, big data) y no genérico.
