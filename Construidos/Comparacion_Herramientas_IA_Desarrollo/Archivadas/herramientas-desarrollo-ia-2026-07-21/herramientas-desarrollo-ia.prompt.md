## Contexto específico — Comparación de Herramientas de IA para Desarrollo

**Referencia**: `referencias/IA/herramientas-desarrollo-ia/HerramientasDesarrollo.jpg` — tabla comparativa de 3 agentes de IA para desarrollo de software: Codex (OpenAI), Claude Code (Anthropic) y OpenCode (open source). Sin branding de terceros que remover — el contenido ya es genérico/comparativo.

**Contenido real (sin inventar cifras), 6 aspectos comparados**:

| Aspecto | Codex (OpenAI) | Claude Code (Anthropic) | OpenCode (Open Source) |
|---|---|---|---|
| Enfoque principal | Agente de desarrollo que trabaja desde la terminal. Lee, modifica y ejecuta código en tu proyecto. | Agente orientado a análisis profundo, razonamiento y comprensión de proyectos grandes. | Plataforma open source y flexible que permite usar diferentes modelos y configuraciones. |
| Fortalezas | Integración simple, ejecución rápida de tareas, ideal para implementación y debugging. | Excelente para refactorización, revisión de arquitectura, explicaciones detalladas y análisis de código. | Personalizable, extensible, soporte para múltiples modelos, comunidad activa. |
| Mejor para | Desarrollo diario, creación de funcionalidades, pruebas, corrección de errores y automatización. | Diseño de arquitectura, revisión de seguridad, comprensión profunda de código y documentación. | Equipos que buscan flexibilidad, control total y adaptar la herramienta a sus necesidades. |
| Facilidad de uso | Muy fácil de instalar y usar. Ideal para empezar rápido. | Fácil de usar, pero requiere contexto claro para aprovechar su potencial. | Requiere más configuración inicial, pero ofrece mayor control y personalización. |
| Modelo de acceso | Requiere cuenta OpenAI (ChatGPT Plus/Pro). | Requiere cuenta Claude (Pro, Team o Enterprise). | Open source. Puedes usar modelos propios (OpenAI, Claude, Gemini, modelos locales, etc.). |
| Cuándo usarla | Cuando necesitas avanzar rápido en el desarrollo y ejecutar tareas concretas. | Cuando necesitas entender a fondo un proyecto, mejorar su diseño o tomar decisiones técnicas importantes. | Cuando quieres flexibilidad total y experimentar con diferentes modelos y flujos de trabajo. |

**Recomendación general de la referencia**: no se trata de usar las tres al mismo tiempo, sino de elegir la adecuada para cada etapa y tipo de tarea — (1) base sólida: usa un buen IDE como VS Code; (2) agente principal: elige tu agente principal (Codex o Claude Code) según tu flujo de trabajo; (3) complemento flexible: usa OpenCode cuando necesites personalización o trabajar con varios modelos.

**Cierre (verbatim)**: "La IA no reemplaza al desarrollador. Lo potencia. La clave está en combinar criterio técnico + buenas herramientas + intención."
