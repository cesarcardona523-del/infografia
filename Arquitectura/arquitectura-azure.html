Arquitectura de Datos en Azure: Patrón Medallón de Extremo a Extremo

De múltiples orígenes crudos a insights de negocio — tres capas de refinamiento progresivo (Bronze → Silver → Gold) con gobernanza integrada.

1 · Data Sources
APIsIntegraciones de terceros
WebhooksEventos push de sistemas externos
IoTTelemetría de dispositivos

2 · Ingestion & Staging

ADLS Gen2Landing zone del data lake
Patrón batch: para tiempo real, el equivalente es Event Hubs + Stream Analytics.
›
3 · Medallion Processing
Bronze — Raw

Datos sin modelar, tal como llegan (JSON/CSV crudos, inmutables).

Spark: valida esquema y tipos
Silver — Cleansed

Datos estructurados, deduplicados y validados.

Spark: agrega métricas de negocio
Gold — Business Ready

Agregados y KPIs, listos para consumo en BI.

CREATE TABLE gold.kpi_ventas AS SELECT region, SUM(monto) AS total FROM silver.ventas GROUP BY region;

5 · Presentation

Ventas por región
Alimentado directamente desde la capa Gold.
Gobernanza de datos — Microsoft Purview cataloga y rastrea el linaje entre capas; cada promoción Bronze→Silver→Gold debe validar esquema y calidad antes de avanzar, para no propagar datos corruptos a los reportes de negocio.
