Automatización de Reportes con Python: de Excel a Producción

Un pipeline ETL real que reemplaza hojas de cálculo manuales por un flujo repetible, validado y auditable.

1 · Origen de Datos
Excel / CSVDatos de ventas u operación diaria, generados manualmente.
Punto de partida: archivo plano, sin validación previa.

2 · Carga Estructurada
pandas.read_excelLee y tipa columnas antes de cualquier transformación.
Falla rápido si el esquema de origen cambia.
›
3 · Procesamiento (Python)
Limpieza y reglas de negocioFiltra, calcula columnas derivadas (ej. Ganancia).
import pandas as pd from sqlalchemy import create_engine import os df = pd.read_excel("datos_entrada.xlsx") df_clean = df[df["Ventas"] > 1000] df_clean["Ganancia"] = df_clean["Ventas"] - df_clean["Costos"] # credenciales vía variable de entorno, no hardcoded engine = create_engine(os.environ["DATABASE_URL"])

4 · Salida
Base de Datos (SQL)Almacenamiento seguro de los datos ya procesados.
Comunicación (Email)Envío automático de alertas y reportes vía SMTP.
Un mismo procesamiento, dos destinos desacoplados.
